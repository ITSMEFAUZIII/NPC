SECTION 1 — engine/HeroAI.js (Sir Aldric's brain)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Aldric is a fully autonomous agent. He runs his own quest
independent of the player. Player can only INFLUENCE him
indirectly — never control him directly.

ALDRIC'S 30-DAY QUEST TIMELINE (data/questScript.js):

Day 01-03 | ARRIVAL
  - Walks into Ashenveil from south road
  - Goes to tavern, sits at bar (ignores Kael)
  - Monologue: "I've heard there's work here."
  - Leaves tavern at dusk, camps outside

Day 04-06 | THE QUEST BEGINS
  - Meets Elder Voss at town well, 09:00
  - Receives quest: find the Sunfire Relic
  - Returns to tavern, announces quest loudly
  - Buys equipment from blacksmith
  - Recruits villager Tom as companion

Day 07-10 | GOBLIN TROUBLES
  - Scouts forest north of village
  - Returns with health -30 (goblin ambush)
  - Stays in tavern 2 days to recover
  - Demands best room (pushes out paying guest)
  - Complains about goblins loudly every evening

Day 11-15 | THE ARTIFACT HUNT
  - Learns relic is in northern dungeon
  - Tells everyone in tavern that evening
  - Recruits a second companion (villager Pip's brother)
  - Studies old map (sits at tavern corner table)

Day 16-20 | INTO THE DUNGEON
  - Leaves village at dawn, Day 16
  - World becomes noticeably quieter
  - Villagers ask Kael where Aldric went
  - Kael has maximum freedom during this period

Day 21-25 | TRIUMPHANT RETURN
  - Returns Day 21 with Sunfire Relic
  - Throws 30 gold to crowd (none to Kael)
  - Celebration in tavern, Kael serves all night
  - Relic gives Aldric +30% strength buff

Day 26-29 | MARCH TO CASTLE
  - Leaves village Day 26 for Dark Lord's castle
  - Villagers line the road, cheer and wave
  - Kael watches from tavern doorway
  - Nobody waves at Kael

Day 30 | THE RECKONING
  - Off-screen battle
  - Outcome = function of world state Kael created
  - Triggers ending calculation

HERO AI PATHFINDING:
  - Simple waypoint system (not full NavMesh)
  - Array of waypoints per day/activity
  - Move toward current waypoint at speed 4
  - If within 1 unit of waypoint: advance to next
  - Avoid buildings: simple bounding box checks
  - Pathfinding blocked? Pick slightly random direction

HERO STAT MODIFIERS (triggered by Kael's actions):

  ACTION: kael_served_poison_drink
    -> hero.health -= 25
    -> hero.health regen disabled for 2 days
    -> Returns from dungeon visibly weakened
    -> questStage advances slower

  ACTION: kael_gave_false_map
    -> hero goes to WRONG dungeon location Day 16
    -> Returns empty-handed Day 20
    -> hero.arrogance -= 20, hero.trust -= 15
    -> questStage delayed by 4 days

  ACTION: kael_spread_negative_rumor
    -> villager.trust[aldric] -= 20
    -> Day 21 celebration smaller
    -> hero.gold received -= 50%

  ACTION: kael_gave_hero_weapon
    -> hero.health maxHealth += 20
    -> Dungeon return faster (+2 days early)
    -> hero.trust += 10
    -> At trust > 60: hero glances at Kael once

  ACTION: kael_warned_village
    -> world.goblinThreat -= 30
    -> Villagers alive for ending
    -> Aldric finds cleared path

PERCEPTION SYSTEM:
  Each tick, chance Aldric notices Kael:
    base_chance = hero.perception / 100 * 0.01 (1% max per tick)
    modified by: kael.stats.influence / 100 * 0.05
    if (Math.random() < chance): hero.noticedKael = true
  
  noticedKael triggers ONCE in entire playthrough:
    - Aldric stops walking
    - Turns to face Kael
    - Holds for 3 seconds
    - Says: "You there... have we met?"
    - Kael can respond: [Yes] [No] [Say nothing]
    - Then Aldric shakes head and walks away
    - This is the most emotional moment in the game

MOVEMENT SPEC:
  Walk speed  : 3.5 units/sec
  Hurry speed : 5.5 units/sec (Shift held)
  Turn speed  : 2.5 rad/sec (with mouse)
  Collision   : simple sphere vs AABB building checks

MOVEMENT LIMITS:
  Cannot leave world bounds (-1000 to 1000)
  Cannot enter castle (locked until Day 26)
  Can enter: tavern, all village buildings, dungeon entrance
  Cannot follow Aldric into dungeon (blocked)

INTERACTION RAYCAST:
  Cast ray from camera center
  Max distance: 2.5 units
  Highlight hit object with outline shader
  Show context prompt: "E - [action]"

CONTEXT ACTIONS (shown based on what is hit):

  Hit: Bar counter
    Options: [1] Serve Drink [2] Clean Bar [3] Check Stock

  Hit: Customer NPC (in tavern)
    Options: [1] Serve [2] Listen [3] Spread Rumor [4] Bribe

  Hit: Sir Aldric
    Options: [1] Serve him [2] Eavesdrop [3] Slip note [4] Stare

  Hit: Trapdoor (discovered)
    Options: [E] Open trapdoor to Black Market

  Hit: Notice board
    Options: [E] Read notices

  Hit: Orphan Pip (village)
    Options: [1] Talk [2] Give coin [3] Listen to stories

  Hit: Villager
    Options: [1] Warn [2] Spread rumor [3] Trade [4] Observe

KAEL'S INTERNAL MONOLOGUE:

  Shows as floating italic text above Kael's head briefly
  Examples:
    "He didn't look at me again today."

SECTION 4 — engine/ConsequenceEngine.js
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Actions don't have immediate effects — they ripple forward in time.
This creates the "butterfly effect" feel.

CONSEQUENCE QUEUE:
  Each Kael action registers a delayed consequence:
  {
    triggerDay: current_day + delay,
    triggerHour: specific_hour,
    action: function(state),
    description: string (for world log)
  }

EXAMPLES:

  ACTION: serve_spiked_drink to Aldric
    Registered consequence: { triggerDay: +1, action: 
      () => { hero.health -= 25, emit('world:event', 
      { description: 'Sir Aldric looks pale today' }) }
    }

  ACTION: spread_rumor (dark lord stronger than reported)
    Registered consequence: { triggerDay: +3, action:
      () => { world.darkLordPower += 15,
              villageHope -= 10,
              emit('world:event', 
              { description: 'Morale drops in the village' }) }
    }

  ACTION: warn_village_about_goblins
    Registered consequence: { triggerDay: +2, action:
      () => { world.goblinThreat -= 30,
              emit('world:event',
              { description: 'Villagers seem more prepared' }) }
    }

  ACTION: give_hero_weapon
    Registered consequence: { triggerDay: +5, action:
      () => { hero.maxHealth += 20,
              if(hero.noticedKael) emit('hero:action', 
              { action: 'grateful_nod' }) }
    }

Each tick: scan queue, execute triggered consequences, remove them.

SECTION 7 — engine/AudioEngine.js
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Positional 3D audio using Howler.js + Web Audio API.

AMBIENT SOUNDS (Web Audio API procedural):
  Tavern ambience:
    - Low pass filtered pink noise (crowd murmur)
    - Random "glass clink" sounds every 5-15 seconds
    - Fireplace crackle: brownian noise + filter
    - Wood creak: occasional low thud
  
  Outdoor ambience:
    - Wind: white noise + dynamic filter based on weather
    - Birds: day only, random chirp intervals
    - Rain: if weather=rain, heavy white noise
    - Night crickets: night only, modulated sine waves

HERO SOUNDS:
  - Heroic hum: subtle musical tone when Aldric nearby
  - Armor clink: on each step
  - Victory fanfare: Day 21 return

KAEL SOUNDS:
  - Footstep: soft thud, wooden floor vs stone different
  - Bottle pour: when serving drink
  - Coin clink: when receiving payment
  - Page turn: journal open/close

MUSIC:
  Procedural ambient music using Web Audio API oscillators:
  
  TAVERN THEME (day):
    - Root: oscillators on C minor pentatonic
    - Slow arpeggios, lute-like timbre
    - Filter envelope gives pluck feel
    - 64-bar generative loop
  
  NIGHT THEME:
    - Same harmony, lower octave
    - More reverb (convolver)
    - Slower, more melancholic

  HERO THEME (when Aldric in scene):
    - Rises slightly in brightness (filter opens)
    - Subtle heroic fanfare motif layered
    - Volume ducks when Aldric leaves

  ENDING MUSIC:
    - 7 different short pieces per ending
    - All variations of same melody
    - Tone matches emotional resolution
